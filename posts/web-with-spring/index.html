Spring으로 게시판 만들기: 기본

December 15, 2020

먼저 여기서 서술된 java project 코드는 Webjjang 무조건 Spring , 웹짱이영환쌤 - YouTube 을 가공한 것임을 밝힌다.

1. Project 생성 및 서버 구동 - pom.xml, ojdbc, 사용자 정의 파일#

Spring Legacy Project 를 생성할 때 Spring MVC Project 를 선택한다. Project 가 생성이 된 후에는 pom.xml 파일을 설정한다. 필자는 웹짱매니아에서 제공한 pom.xml 파일을 수정해 사용했다. pom.xml 은 코드가 길기 때문에 글 맨 아래에 두었다. 복사 붙여넣기 할 때는 version 태그 아래부터 하자.

ojdbc6.jar 와 사용자 정의 파일 pageObject.jar 을 src/main/webapp/WEB-INF/lib 에 넣어준다. lib 가 없으면 폴더를 생성한다. 이렇게 파일을 디렉토리에 추가한다고 Spring 에서 인식하는 것은 아니다. pom.xml 에서 아래와 같이 설정해야 한다. <systemPath> 를 보면 디렉토리를 알 수 있다.

pom.xml 설정

<dependency>
    <groupId>net.webjjang</groupId>
    <artifactId>util.pageObject</artifactId>
    <version>1.0.0</version>
    <scope>system</scope>
    <systemPath>${project.basedir}/src/main/webapp/WEB-INF/lib/pageObject.jar</systemPath>
</dependency>

<dependency>
    <groupId>oracle</groupId>
    <artifactId>ojdbc6</artifactId>
    <version>11.2.0</version>
    <scope>system</scope>
    <systemPath>${project.basedir}/src/main/webapp/WEB-INF/lib/ojdbc6.jar</systemPath>
</dependency>

성공적으로 반영되면 Maven Dependencies 에 ojdbc 와 사용자 정의 파일이 등록된 것을 알 수 있다.

그리고 java version 을 맞춰야 한다. 기존의 pom.xml파일을 복사 붙여넣기 할 경우, <java-version> 를 보면 1.8 이라고 된 경우가 있다. Project 의 버전 역시 이것으로 맞춰야 한다. 실제로 Eclipse 에서 해당 project 의 Properties 에서 Project Facets 를 확인해보면 java 1.6 이라고 돼 있을 있다. 이것을 변환하기 위해 해당 project 의 Maven 에서 Update project 를 실행한다. 여기서 Update project configuration from pom.xml 을 체크한 후에 ok 를 클릭한다. 이제 다시 Project Facets 를 확인하면 java 1.8 로 변한 것을 확인할 수 있다.

그 다음은 tomcat 설정인데, 해당 spring project 를 tomcat server 에 놓자. 그 다음에 서버를 실행해보자. 그런 다음 http://localhost:PORT/main/ 에 정상적으로 접속되면 서버가 제대로 작동하는 것이다.

기타 과정으로, server operation 시간 제한을 설정해보자. Tomcat server 를 더블 클릭한 후 Overview > Timeouts 에서 start 를 180 으로 설정한다.

2. URL 에서 Java Project name 제거#

두 가지 방법이 있다. 첫 번째는 tomcat 서버 설정을 변경하는 것이다. Tomcat server 를 더블 클릭한 후 modules 탭을 클릭한다. 그리고 변경하고자 하는 프로젝트를 선택한 후 Edit 버튼을 누른다. /main 을 / 으로 고치고 OK 를 클릭한다. Ctrl + S 를 눌러 저장해주면 된다.

두 번째 방법은, project 설정을 변경하는 방법이다. Eclipse 에서 해당 project 를 우클릭하여 properties 에 들어간다. 여기서 Web Project Settings 를 클릭한다. 그리고 Context root 를 /main 에서 / 으로 고친다. 그리고 project clean 을 한 후에, 서버를 삭제하고 다시 추가한다.

3. Encoding UTF-8 설정 및 default Servlet - web.xml, jsp#

이제 웹페이지에서 unicode 가 제대로 나오게 하기 web.xml 에서 encoding 을 utf-8 으로 설정한다. src/main/webapp/WEB_INF/web.xml 에 아래의 코드를 추가한다.

web.xml 추가 설정

<filter>
    <filter-name>encodingFilter</filter-name> 
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class> 
    <init-param> 
       <param-name>encoding</param-name> 
       <param-value>UTF-8</param-value> 
    </init-param> 
    <init-param> 
       <param-name>forceEncoding</param-name> 
       <param-value>true</param-value> 
    </init-param> 
 </filter> 
 <filter-mapping> 
    <filter-name>encodingFilter</filter-name> 
    <url-pattern>/*</url-pattern> 
 </filter-mapping>

그리고 home.jsp 파일에 아래와 같은 코드를 추가한다.

<%@ page contentType="text/html; charset=UTF-8" pageEncoding="UTF-8" language="java" %>

참고로 default web.xml 설정을 살펴보자.

default web.xml

<!-- The definition of the Root Spring Container shared by all Servlets and Filters -->
<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>/WEB-INF/spring/root-context.xml</param-value>
</context-param>

<!-- Creates the Spring Container shared by all Servlets and Filters -->
<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>

<!-- Processes application requests -->
<servlet>
    <servlet-name>appServlet</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <init-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>/WEB-INF/spring/appServlet/servlet-context.xml</param-value>
    </init-param>
    <load-on-startup>1</load-on-startup>
</servlet>
    
<servlet-mapping>
    <servlet-name>appServlet</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>

</web-app>

크게 <context-param> 과 <servlet> 가 있다. <context-param> 는 root-context.xml 을, <servlet> 은 servlet-context.xml 을 <param-value> 로 갖고 있는데, root-context.xml 는 non-web 을 servlet-context.xml 은 web 의 bean 을 설정한다.

<servlet>, <servlet-mapping> 두 가지 중 <servlet-mapping> 설정이 의미하는 것은 <url-pattern> 에 해당하는 url 을 <servlet-name> 이 appServlet 인 servlet 이 처리한다는 의미이다. <servlet> 에는 servlet 이 정의돼 있다. 위에서는 DispatcherServlet class 이다. 그리고 이 servlet 의 세부 설정은 servlet-context.xml 에 있다.

4. Servlet 설정 확인 - servlet-context.xml, Controller#

web.xml 에서 servlet 설정을 확인했다. 세부 설정은 servlet-context.xml 에서 확인할 수 있다. Default 는 아래와 같다.

Default servlet-context.xml

<!-- Handles HTTP GET requests for /resources/** by efficiently serving up static resources in the ${webappRoot}/resources directory -->
    <resources mapping="/resources/**" location="/resources/" />

    <!-- Resolves views selected for rendering by @Controllers to .jsp resources in the /WEB-INF/views directory -->
    <beans:bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
        <beans:property name="prefix" value="/WEB-INF/views/" />
        <beans:property name="suffix" value=".jsp" />
    </beans:bean>
    
    <context:component-scan base-package="com.mycompany.main" />

<resources> 는 html, css, javascript, image 등을 mapping 하기 위한 설정이다. <beans> 는 Controller 가 return 할 때 사용하는 class, prefix, suffix 를 설정한다.

아래는 default HomeController 에서 home method 이다.

Default HomeController home method

@RequestMapping(value = "/", method = RequestMethod.GET)
    public String home(Locale locale, Model model) {
        logger.info("Welcome home! The client locale is {}.", locale);
        
        Date date = new Date();
        DateFormat dateFormat = DateFormat.getDateTimeInstance(DateFormat.LONG, DateFormat.LONG, locale);
        
        String formattedDate = dateFormat.format(date);
        
        model.addAttribute("serverTime", formattedDate );
        
        return "home";

“home” 이 return 된다. 하지만 servlet-context.xml 의 설정에 의해 InternalResourceViewResolver 가 prefix + “home” + suffix 를 return 한다. 즉 /WEB-INF/views/home.jsp 를 return 한다.

5. Controller 설정 1: 생성 - servlet-context.xml, @Controller#

HomeController 외에 다른 Controller 가 필요할 경우 설정한다. 예시는 ForumController 를 생성하는 경우다. src/main/java > com.“mycompany”.forum.controller 패키지를 만든다. 그 패키지 하위에 ForumController 라는 Class 를 생성한다. 생성한 후에 @Controller 를 입력해 annotation 처리를 한다.

ForumController

import org.springframework.stereotype.Controller;
@Controller
public class ForumController {
}

@Controller 는 @Component 를 상속한 annotation 으로서 class ForumController 의 객체를 메모리에 생성하며, DispatcherServlet 이 인식할 수 있도록 Controller 의 객체로 만들어준다 (채규태, 2020).

ForumController 를 Spring 의 제어를 받도록 설정해줘야 한다. src > main > WEB-INF > spring > appServlet > servlet-context.xml 을 연다.

<context:component-scan base-package="com.mycompany.main" />

라고 돼 있는 것을 아래와 같이 수정한다.

<context:component-scan base-package="com.mycompany" />

변경 사항이 적용되면 Eclipse Package Explorer 의 ForumController 에 S 마크가 붙는다.

6. Controller 설정 2: request - Model, @RequestMapping, @GetMapping, .do#

ForumController 에 @RequestMapping 을 추가한다.

ForumController

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.ui.Model;
import net.webjjang.util.PageObject;
@Controller
@RequestMapping("/forum")
public class ForumController {

    private final String MODULE_NAME = "forum";
    // 추후 설명
    @Autowired
    private ForumService service;
    @GetMapping("/list.do")
    public String list(Model model, PageObject pageObject) {
        System.out.println("ForumController.list().pageObjet:" + pageObject);

        model.addAttribute("list", service.list(pageObject));
        model.addAttribute("pageObject", pageObject);
        return MODULE_NAME + "/list";
    }
}

@RequestMapping 은 클라이언트의 “/forum” request 에 method 를 대응한다는 의미다 (채규태, 2020). 위 코드에서는 class 를 매핑했다.

DB에서 data 를 가져와서 jsp 로 보낼 때는 request 객체가 필요하다. request 객체는 Model 객체에 담겨 있다. PageObject 는 사용자가 정의한 jar 파일이다.

@GetMapping 은 get 방식 request 에 대응하는 @RequestMapping 이다. 위의 코드에서 @GetMapping("/list.do") 가 의미하는 것은 /list.do 에서 request 를 받겠다는 것이다. 즉, request 를 받는 url 을 임의로 지정할 수 있다. 예제에서는 .do 로 했는데, .abc, .zzz 등 어떻게 설정해도 상관 없다.

또한 @RequestMapping("/forum") 가 class 에 위치하고, @GetMapping("/list.do") 은 method 에 위치하기 때문에 request url 은 “/forum/list.do” 다.

7. Mybatis 설정: root-context.xml#

Mybatis 프레임워크의 특징은 두 가지로, 첫째는 한두 줄의 자바 코드로 DB 연동을 처리한다는 것이며, 둘째는 SQL 명령어를 자바 코드에서 분리하여 xml 파일에 따로 관리한다는 것이다 (채규태, 2020).

root-context.xml 파일을 설정한다. 아래는 웹짱매니아에서 제공한 root-context.xml 파일을 수정한 것이다.

root-context.xml

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:context="http://www.springframework.org/schema/context"
    xmlns:mybatis-spring="http://mybatis.org/schema/mybatis-spring"
    xsi:schemaLocation="http://mybatis.org/schema/mybatis-spring http://mybatis.org/schema/mybatis-spring-1.2.xsd
  http://www.springframework.org/schema/beans https://www.springframework.org/schema/beans/spring-beans.xsd
  http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context-4.3.xsd">

    <!-- Root Context: defines shared resources visible to all other web components -->
    <bean id="hikariConfig" class="com.zaxxer.hikari.HikariConfig">
        <property name="driverClassName"
            value="oracle.jdbc.driver.OracleDriver" />
        <property name="jdbcUrl"
            value="jdbc:oracle:thin:@localhost:1521:xe" />
        <!-- value="jdbc:oracle:thin:@IP_ADDRESS:PORT:SID" -->
        <!-- 
        <property name="jdbcUrl"
            value="jdbc:log4jdbc:oracle:thin:@402-oracle:1521:orcl" />
        -->

        <property name="username" value="USERNAME" />
        <property name="password" value="PASSWORD" />
    </bean>
    <bean id="dataSource" class="com.zaxxer.hikari.HikariDataSource"
        destroy-method="close">
        <constructor-arg ref="hikariConfig" />
    </bean>
    <!-- Mybaits - dao 코드 자동 생성 -->
    <!-- mapper Interface(메서드이름) - mapper xml(sql) -->
    <bean id="sqlSessionFactory"
        class="org.mybatis.spring.SqlSessionFactoryBean">
        <property name="dataSource" ref="dataSource"></property>
    </bean>

    <!-- Mapper 인식 -->
    <mybatis-spring:scan base-package="com.mycompany" />

    <context:component-scan base-package="com.mycompany">
    </context:component-scan>

</beans>

설정이 완료되면 Eclipse 에서 root-context.xml 의 Namespaces 를 확인한다. Namespaces 에 체크 된 항목들은 root-context.xml 의 <beans> 의 xmlns 항목임을 알 수 있다. 또한 <property> 설정과 <constructor-arg> 설정이 있는데, 이는 각각 Setter 와 Constructor 를 이용해 Dependency Injection 을 처리한다.

<bean id=“sqlSessionFactory> 를 보자. SqlSessionFactoryBean class 를 사용하는 bean 인데, 사용 방법에는 두 가지가 있다. 첫째는 DAO 를 작성하는 방법, 두 번째는 mapper interface 를 사용하는 방법이다. 여기서는 mapper interface 를 사용한다.

Mapper interface 를 사용하면 사용자가 DAO 를 만들지 않고 mybatis 가 DAO 를 생성한다. <mybatis-spring:scan> 은 base-package 에서 mapper interface 를 찾는 설정이다.

<context:component-scan> 와 헷갈리지 말자. <context:component-scan> 는 애플리케이션에서 사용할 객체들을 <bean> 에 등록하지 않고도 스프링 설정 파일에 자동으로 생성하기 위한 엘리먼로서, 이 설정을 추가하면 스프링 컨테이너는 클래스 패스에 있는 클래스들을 스캔하여 @Component 가 설정된 클래스들을 자동으로 객체 생성한다 (채규태, 2020).

8. Mybatis SQL Mapper XML 설정: ForumMapper.xml (SQL Mapper XML), ForumMapper (interface)#

SQL Mapper XML 은 SQL 명령어가 저장되는 파일로, 필자는 웹짱매니아에서 제공한 boardMapper.xml 파일을 수정해 ForumMapper.xml 로 사용했다. 이 파일을 src/main/resources 하위에 두어야 한다.

하위 디렉토리는 반드시 <mapper> 의 namespace 와 동일하게 하자. 만약 namespace 와 다른 디렉토리에 ForumMapper.xml 위치하면 다음과 같은 에러가 발생한다.

Request processing failed; nested exception is org.apache.ibatis.binding.BindingException: Invalid bound statement (not found): com.webjjang.forum.mapper.ForumMapper.getCount

사실 namespace 는 ForumMapper.java Interface 의 경로이면서, ForumMapper.xml 의 경로이다. 아래 예시에서는 namespace 가 com/webjjang/forum/mapper/ForumMapper 로 돼 있으니, src/main/resources/com/webjjang/forum/mapper 폴더를 생성한 후에, Eclipse 를 통해 여기에 ForumMapper.xml 파일을 생성한 후 아래와 같이 작성한다. ForumMapper.xml 에서 id 는 ForumMapper.java 의 method 명과 동일해야 한다.

ForumMapper.xml

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
"http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.webjjang.forum.mapper.ForumMapper">
    <select id="list" resultType="com.webjjang.forum.vo.ForumVO">
        select * from (
        select rownum rnum, no, title, writer, writeDate, hit
        from (
        select no, title, writer, writeDate, hit
        from forum
        <!-- 이곳에 조건을 쓴다. -->
        <include refid="search"></include>
        order by no desc
        )
        )
        where rnum between #{startRow} and #{endRow}
    </select>
    <select id="getCount" resultType="java.lang.Integer">
        select count(*)
        from forum
        <!-- 이곳에 조건을 쓴다. -->
        <include refid="search"></include>
    </select>

    <insert id="write">
        insert into forum(no, title, content, writer, pw)
        values(forum_seq.nextval, #{title}, #{content}, #{writer}, #{pw})
    </insert>

    <sql id="search">
        <!-- 검색어가 존재하는 경우에만 검색을 실행한다. mybatis doc을 참조하시오. -->
        <if test="word != null and word != ''.toString() ">
            where ( 2 = 1 <!-- 형식을 맞추기 위한 false를 넣었다. or 연산시 하는 기법 and 연산시 하는 기법은 true를 
                넣기다. -->
            <!-- Mybatis에서 2글자 이상인 경우 : String 취급, 한 글자인 경우:char 취급 => toString()를 
                이용해서 문자열 String로 만듬 -->

                or content like '%' || #{word} || '%'

            </if>
            <if test="key == 'tc'.toString()">
                or title like '%' || #{word} || '%'
                or content like '%' ||
                #{word} || '%'

                or title like '%' || #{word} || '%'

9. Mapper Interface 생성 및 사용: ForumMapper.xml (SQL Mapper XML), ForumMapper (interface), ForumVO, ForumService, @Inject#

ForumMapper.xml 의 namespace 는 mapper interface 의 경로라고 위에서 언급했다. src/main/java 에 namespace 에 해당하는 경로에 interface 를 생성한다. 여기서는 ForumMapper 를 아래와 같이 생성했다. ForumMapper 의 method 명은 ForumMapper.xml 의 id 와 동일해야 한다.

ForumMapper

package com.webjjang.forum.mapper;
import java.util.List;
import com.webjjang.forum.vo.ForumVO;
import net.webjjang.util.PageObject;
public interface ForumMapper {
    // 게시판 리스트
    public List<ForumVO> list(PageObject pageObject);
    public Integer getCount(PageObject pageObject);
    // 게시판 글 보기
    public ForumVO view(int no);
    public void increase(int no);
    // 게시판 글 작성
    public Integer write(ForumVO vo);

이제 ForumService class 를 src/main/java 에 아래와 같이 생성하자. 보통은 ForumService 를 interface로 만든 후에 ForumServiceImpl을 만들어 상속한다. 하지만 여기서는 단일 class 로 생성한다. 참고로 Service 는 DB 데이터를 DAO 로부터 전달받아 가공하는 것으로, 여기서는 mapper interface 가 DAO 의 역할을 한다.

ForumService

package com.webjjang.forum.service;
import java.util.List;
import javax.inject.Inject;
import org.springframework.stereotype.Service;
import com.webjjang.forum.mapper.ForumMapper;
import com.webjjang.forum.vo.ForumVO;
import net.webjjang.util.PageObject;
@Service
public class ForumService {
    @Inject
    private ForumMapper mapper;
    // 게시판 글 보기
    public List<ForumVO> list(PageObject pageObject) {
        // paeObject에 전체 데이터의 개수를 셋팅하는 것이 필요
        pageObject.setTotalRow(mapper.getCount(pageObject));
        return mapper.list(pageObject);
    }
    public ForumVO view(int no, int inc) {
        if(inc == 1) {
            mapper.increase(no);
        }
        return mapper.view(no);
    }
    // 검색 게시판 글 수정
    public void update(ForumVO vo) throws Exception {
        if(mapper.update(vo) == 0) {
            throw new Exception("입력한 정보를 확인해주세요.");
        }
    }
    public void write(ForumVO vo) throws Exception {
        if(mapper.write(vo) == 0) {
            throw new Exception("입력한 정보를 확인해주세요.");
        }
    }
    public void delete(ForumVO vo) throws Exception{
        if(mapper.delete(vo) == 0) {
            throw new Exception("입력한 정보를 확인해주세요.");
        }
    }   
}

@Inject 는 자동으로 Dependency Injection 을 실행하는 annotation 중 하나다. Dependency Injection 을 자동 실행하는 annotation 은 @Autowired, @Inject, @Resource 이다.

@Autowired 는 스프링에서만 사용할 수 있지만, @Inject 와 @Resource 는 JSR 에 실려있는 자바 기존의 annotion 으로 스프링 외에서도 사용할 수 있다. Dependency Injection 방식에서도 차이가 있는데, @Autowired 와 @Inject 는 타입에 맞춰서 하는 반면, @Resource 는 이름에 맞춰서 연결한다 (쿠우v - OKKY, 2017).

ForumController

package com.webjjang.forum.controller;
import java.net.URLEncoder;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import com.webjjang.forum.service.ForumService;
import com.webjjang.forum.vo.ForumVO;
import net.webjjang.util.PageObject;
@Controller
@RequestMapping("/forum")
public class ForumController {
    private final String MODULE_NAME = "forum";
    @Autowired
    private ForumService service;
    // 게시글 리스트
    @GetMapping("/list.do")
    public String list(Model model, PageObject pageObject) {
        System.out.println("ForumController.list().pageObjet:" + pageObject);
        // DB에서 데이터 가져오기 - Controller - Service - Mapper (DAO 자동 생성)
        model.addAttribute("list", service.list(pageObject));
        model.addAttribute("pageObject", pageObject);
        return MODULE_NAME + "/list";
    }
    // 게시글 보기
    @GetMapping("/view.do")
    public String view(Model model, int no, int inc) {
        model.addAttribute("vo", service.view(no, inc));
//      model.addAttribute("pageObject", pageObject);
        return MODULE_NAME + "/view";
    }
    // 글 쓰기 폼
    @GetMapping("/write.do")
    public String writeForm() {
        return MODULE_NAME + "/write";
    }
    // 글 쓰기 처리 - 제목, 내용, 작성자, 비밀번호
    @PostMapping("/write.do")
    public String write(ForumVO vo) throws Exception{
        System.out.println("ForumController.write().vo:"+vo);
        service.write(vo);
        // 화면에 JSP를 보여주지 않고 바로 list로 이동한다.
        return "redirect:list.do";
    }
    // 글 수정 폼
    @GetMapping("/update.do")
    public String updateForm(Model model, int no, int inc) {
        model.addAttribute("vo", service.view(no, inc));
        return MODULE_NAME + "/update";
    }
    // 글 수정 처리 - 글번호, 제목, 내용, 작성자, 확인용 비밀번호
    @PostMapping("/update.do")
    public String update(ForumVO vo, PageObject pageObject) throws Exception{
        System.out.println("ForumController.update().vo:"+vo);
        System.out.println("ForumController.update().pageObject:"+pageObject);
        // DB 처리
        service.update(vo);
        // 검색어가 null이면 null이 아닌 ""로 변경
        if(pageObject.getWord() == null) pageObject.setWord("");
        // 화면에 JSP를 보여주지 않고 바로 글 보기로 이동한다.
        return "redirect:view.do?no=" + vo.getNo() + "&inc=0" 
        + "&page=" + pageObject.getPage()
        + "&perPageNum=" + pageObject.getPerPageNum()
        + "&key=" + pageObject.getKey()
        // 검색어 한글 처리
        + "&word=" + URLEncoder.encode(pageObject.getWord(), "utf-8");
    }
    // 글 삭제 폼
    @GetMapping("/delete.do")
    public String deleteForm() {
        return MODULE_NAME + "/delete";
    }   
    // 글 삭제 처리
    @PostMapping("/delete.do")
    public String delete(ForumVO vo) throws Exception{
        service.delete(vo);
        return "redirect:list.do";
    }
}

@Autowired 는 자동으로 Dependency Injection 을 실행하는 annotation 중 하나다. Dependency Injection 을 자동 실행하는 annotation 은 @Autowired, @Inject, @Resource 이다.

@Autowired 는 스프링에서만 사용할 수 있지만, @Inject 와 @Resource 는 JSR 에 실려있는 자바 기존의 annotion 으로 스프링 외에서도 사용할 수 있다. Dependency Injection 방식에서도 차이가 있는데, @Autowired 와 @Inject 는 타입에 맞춰서 하는 반면, @Resource 는 이름에 맞춰서 연결한다 (쿠우v - OKKY, 2017).
