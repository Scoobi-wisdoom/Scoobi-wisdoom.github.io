Java의 Constructor 주의 사항

  • Jul 28, 2020
  • One minute read

고용노동부의 지원을 받아 IT 교육을 받고 있다. 지금은 Java를 배우고 있는데, 코딩을 하다가 constructor에 대해 알게 된 것을 쓴다.

혹자는 constructor는 method가 아니라고 하고, 또 다른 사람들은 constructor는 ‘인스턴스 초기화 메서드’라고 한다. 즉 method라고 한다.

constructor가 method가 아니더라도, method와 비슷한 결과를 낼 때가 있다.

public class Login {
	

	
	Frame frame;
	TextField txtid,txtpwd;
	Label label;
	Button btnlogin;
	Button btnsignup;
	Db<String, User> userDb;
	
	public Login() {
		// Caution below
		Db<String, User> userDb = new UserDb("127.0. 0.1", "db", "db");
		makeUi();
	}
}

위의 코드에서 Caution below라고 쓴 부분을 보자. userDb를 Db<String, User>로 초기화 하기 때문에 언뜻 보면 맞는 코드로 보인다.

물론 필자가 이제 막 Java를 배우기 시작했기 때문에 필자만 헷갈렸을 수도 있다. 하지만 위의 코드는 틀렸으며, 아래와 같이 고쳐야 한다.

public Login() {
		// Caution below
//		Db<String, User> userDb = new UserDb("192.168.0.31", "db", "db");
		userDb = new UserDb("127.0. 0.1", "db", "db");
		makeUi();
	}

그 이유는 Db<String, User>를 사용해서 userDb를 초기화할 경우, userDb는 local variable이 되기 때문이다. 그렇게 되면, global에서는 사용할 수 없다.

이는 constructor를 method라고 생각하면 쉽게 이해할 수 있다. Method 내의 local variable은 method 안에서만 사용 가능하다는 것은 모두가 알 것이다.
